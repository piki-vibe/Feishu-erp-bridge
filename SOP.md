# 飞书-金蝶数据同步系统操作SOP（财务版）

## 1. 文档目的
本SOP用于指导财务同事日常使用“飞书-金蝶数据同步系统”，完成任务配置、测试、执行、停止、结果核对与常见问题处理。

## 2. 适用范围
- 使用对象：财务专员、财务主管、业务支持人员
- 业务场景：付款单/收款单等从飞书多维表同步到金蝶

## 3. 使用前准备
- 已获取系统访问地址（如 `https://erp.keyupan.cn`）
- 已有系统账号（如管理员分配）
- 已准备飞书参数：`App ID`、`App Secret`、`App Token`、`Table ID`（可选 `View ID`）
- 已准备金蝶参数：API地址、用户名、密码、账套ID（DB ID）、表单ID

## 4. 首次配置（每个任务独立配置）
说明：每个任务配置互相独立，请按任务逐个配置。

### 4.1 登录系统
1. 打开系统地址。
2. 输入账号密码登录。

### 4.2 新建任务
1. 进入“任务”页面。
2. 点击“新建任务”。
3. 填写任务名称（必填），可填写任务描述。
4. 点击保存。

### 4.3 进入任务配置
1. 在任务列表中找到目标任务。
2. 点击“配置”按钮，进入“任务配置”弹窗。

### 4.4 配置飞书参数
1. 在“飞书基础参数”中填写：
   - `App ID`
   - `App Secret`
   - `App Token`
   - `Table ID`
   - `View ID`（可选）
2. 点击“更新字段列表”。

### 4.5 配置“查询参数字段”（变量映射）
1. 在“查询参数字段”中点击“添加字段参数”。
2. 逐项配置：
   - 变量名：如 `A`、`B`、`C`
   - 字段名：从下拉中选飞书字段
   - 处理方式：优先选“自动”
   - 格式：仅特定字段需要（如数字保留小数位、日期格式）
3. 如需核对字段值，点击该行右侧“查看”：
   - 作用：查看多维表第一条记录该字段的值（仅用于快速查看）
   - 注意：此“查看”不作为最终筛选测试结果依据

### 4.6 配置“筛选字段”
1. 在“筛选字段”中点击“添加筛选条件”。
2. 配置字段名、操作符和值。
3. 推荐配置“同步状态 != 成功”，避免重复同步。

### 4.7 配置“回传字段”
1. 在“回传字段”中点击“添加回写字段”。
2. 常用回写项建议：
   - 同步状态（成功/失败）
   - 错误信息
   - 单据编号
   - 完整响应（可选）

### 4.8 配置金蝶参数与数据模板
1. 填写金蝶登录参数：用户名、密码、DB ID、API地址等。
2. 填写“表单ID”。
3. 在“数据模板(JSON)”中配置请求体。
4. 点击“导入变量”，插入 `{{A}}`、`{{B}}` 等变量。
5. 点击“保存配置”。

## 5. 验证测试（上线前必做）
在任务卡片点击“验证”，按顺序执行：
1. 飞书登录测试
2. 飞书字段查询/筛选/回传测试
3. 金蝶登录测试
4. 第一条记录完整流程测试

关键说明（非常重要）：
- 第4项“第一条记录完整流程测试”会经过筛选条件。
- 第4项只对“筛选后第一条记录”跑完整流程（会真实写入金蝶/飞书）。
- “执行任务”按钮则会对“筛选后全部记录”跑完整流程。

## 6. 日常执行步骤
1. 确认任务状态“已配置”且验证通过。
2. 点击“执行”。
3. 到“执行监控”查看进度、成功数、失败数。
4. 结束后点击“查看详情”核对结果。
5. 如需查看WebAPI明细，打开“WebAPI调用详情（第一条记录）”。

## 7. 停止任务规则
1. 任务运行中可点击“停止”。
2. 系统为“安全停止”机制：先完成当前并发中的记录，再停止。
3. 状态会先显示“停止中”，稍后变为最终状态。

## 8. 常见问题与处理

### 8.1 报错“字段XXX是必填项”
处理：
1. 检查该字段在模板中是否有变量。
2. 检查变量是否已在“查询参数字段”正确映射。
3. 点击“查看”确认该字段值是否为空。
4. 如字段允许为空，请确认模板中该变量允许空值替换。

### 8.2 提示“模板变量未成功替换”
处理：
1. 检查模板里是否存在 `{{变量名}}` 拼写错误。
2. 检查变量名与“查询参数字段”中的变量名是否一致。
3. 检查筛选后的记录该字段是否为空。

### 8.3 第4项测试失败但平时可导入
处理：
1. 确认第4项是“筛选后第一条记录”。
2. 到“执行详情”查看失败字段和返回信息。
3. 用同一筛选条件检查第一条数据是否满足金蝶必填规则。

### 8.4 停止后界面未立即刷新
处理：
1. 先等待几秒（安全停止有收尾过程）。
2. 仍未更新时手动刷新页面后再看状态。

## 9. 财务操作红线
- 不要在生产任务中随意关闭筛选条件。
- 第4项测试会真实写入，测试前先确认数据可落账。
- 任务名与用途要清晰，避免误点执行。
- 非管理员不要改他人任务配置。

## 10. 建议命名规范
- 任务名称建议：`业务类型_来源表_目标单据`
- 示例：`付款单_飞书台账_金蝶付款申请单`

## 11. 每日操作检查清单（建议打印）
- 任务是否启用正确
- 筛选条件是否正确
- 当日是否先做过验证测试
- 执行后成功/失败数量是否符合预期
- 失败记录是否已回写错误信息

## 12. 金蝶 API 方法填写规范（2026-03 更新）

### 12.1 API 方法怎么填

1. `API 方法` 就填金蝶 SDK 里 `client.` 后面的实际方法名。
2. 系统不会自动帮你把一个方法改成另一个方法。
3. 常用示例：
   - 保存：`Save`
   - 删除：`Delete`
   - 查看：`View`
   - 暂存：`Draft`
   - 提交：`Submit`
   - 审核：`Audit`
   - 反审核：`UnAudit`
   - 禁用/反禁用/其他操作：`ExcuteOperation`
   - 撤销：`CancelAssign`
   - 分配：`Allocate`
   - 取消分配：`CancelAllocate`
4. 如果有其他金蝶方法，也可以直接手工输入。

### 12.2 opNumber 怎么填

1. `opNumber` 是可选项。
2. 有值就传给金蝶。
3. 留空就不传。
4. 一般在 `API 方法 = ExcuteOperation` 时使用，例如禁用、反禁用等操作。

### 12.3 查看请求数据怎么看

1. 点击“查看传入数据（不发送）”时，页面会展示完整请求参数。
2. 重点核对以下内容：
   - `requestUrl`
   - `apiMethod`
   - `opNumber`（如果有）
   - `payload.formid`
   - `payload.data`
3. 如果预览正确但执行失败，优先检查：
   - `API 方法` 拼写是否正确
   - `表单ID` 是否正确
   - `data` 里的必填字段是否齐全
   - `Numbers` / `Ids` 是否符合金蝶接口要求
